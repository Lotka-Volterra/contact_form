# 管理ページ 詳細仕様

まだ実装前の機能。実装に着手する前の仕様書として先にまとめておく。CLAUDE.mdの概要を踏まえた詳細版。

## 画面構成

- 一覧画面: `contacts`テーブルの内容を一覧表示する
- 詳細画面: 一覧の項目をクリックすると、その問い合わせの詳細（名前・メールアドレス・件名・本文・ステータス・受付日時）を表示する
- ステータス変更: 詳細画面（または一覧画面）でドロップダウンリストから3段階のステータスを変更できる

## ルーティング方針

管理ページのルートはすべて`/admin`配下にまとめる（例: `GET /admin`で一覧、`GET /admin/:id`で詳細、`POST /admin/:id/status`でステータス更新、など具体的なパスは実装時に確定する）。

理由: 認証（ログイン）は今回のスコープ外だが、将来追加する際に`/admin`配下のルーター全体へ認証ミドルウェアを1箇所差し込むだけで済むようにするため。詳細は[docs/decisions.md](./decisions.md)を参照。

## ステータスの内部コードと表示ラベル

DBの`status`列は英語コードで保持する（[docs/contact-form.md](./contact-form.md)参照）。管理ページの表示・ドロップダウンでは、以下の対応で日本語ラベルに変換する。

| 内部コード（DB上の値） | 表示ラベル（画面上） |
| ----------------------- | --------------------- |
| `new`                    | 新規                   |
| `in_progress`            | 対応中                 |
| `resolved`               | 解決済み               |

この対応表はコード上でも一箇所（例: `services/`配下）にまとめ、複数箇所に同じマッピングを重複して書かないようにする。

## 認証

今回のスコープ外。管理ページは誰でもアクセスできる状態のまま実装する。将来の追加方針は[docs/decisions.md](./decisions.md)を参照。
